# Q23. 배치 정책 — First/Next/Best Fit 실전 트레이드오프

> CSAPP 9.9 | 8부. malloc | 심화

## 질문

1. 세 가지 배치 정책(First Fit, Next Fit, Best Fit)의 시간 복잡도와 단편화 경향을 비교하시오.
2. [검증] 가용리스트 [32B, 64B, 24B, 128B]에서 malloc(20)(=24B 필요) 시 각 정책이 선택하는 블록과 split 잔여 크기를 구하시오.

## 답변

### 최우녕

> 세 가지 배치 정책(First Fit, Next Fit, Best Fit)의 시간 복잡도와 단편화 경향을 비교하시오.

```text
━━━ First Fit ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  방법: free list의 처음부터 탐색해서 처음 맞는 블록을 선택
  시간: O(가용 블록 수) — 최악의 경우 전체 리스트 순회
  단편화:
  ㄴ 리스트 앞쪽에 작은 조각이 쌓이는 경향 (앞쪽을 계속 쪼개니까)
  ㄴ 뒤쪽에는 큰 블록이 남아있을 가능성
  ㄴ 시간이 지나면 앞쪽 탐색이 느려진다

━━━ Next Fit ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  방법: 마지막으로 탐색을 멈춘 위치부터 이어서 탐색
  시간: O(가용 블록 수) — 최악은 같지만 평균적으로 First Fit보다 빠르다
  단편화:
  ㄴ 조각이 리스트 전체에 고르게 퍼진다
  ㄴ 큰 블록도 쪼개지기 쉬워서 First Fit보다 외부 단편화가 더 심할 수 있다
  ㄴ 연구 결과 실전에서 First Fit보다 메모리 이용률이 낮다

━━━ Best Fit ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  방법: 전체 free list를 탐색해서 요청 크기에 가장 가까운 블록을 선택
  시간: O(가용 블록 수) — 항상 전체 순회 (단, segregated list에서는 ~O(1))
  단편화:
  ㄴ split 후 잔여 조각이 가장 작다 → 내부 단편화 최소
  ㄴ 하지만 아주 작은 조각이 많이 생김 → 쓸모없는 외부 단편화 발생 가능
  ㄴ 종합적으로 메모리 이용률은 가장 좋은 편

━━━ 비교 요약 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  정책       | 탐색 속도  | 단편화     | 실전 평가
  -----------|-----------|-----------|------------------
  First Fit  | 중간      | 앞쪽 집중  | 단순하고 무난
  Next Fit   | 빠름      | 고르게 나쁨 | 큰 블록 보존 못함
  Best Fit   | 느림      | 가장 적음  | segregated list와 결합 시 최적
```

> [검증] 가용리스트 [32B, 64B, 24B, 128B]에서 malloc(20)(=24B 필요) 시 각 정책이 선택하는 블록과 split 잔여 크기를 구하시오.

```text
  전제: 최소 블록 크기 = 16B (헤더 8B + 최소 페이로드 8B)
  요청: malloc(20) → 정렬 후 필요한 블록 크기 = 24B

  가용리스트 순서: [32B] → [64B] → [24B] → [128B]

━━━ First Fit ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  처음부터 탐색 → 32B >= 24B → 32B 선택
  |
  ㄴ 잔여 = 32 - 24 = 8B
  ㄴ 8B < 16B(최소 블록) → split 불가
  ㄴ 32B 전체 할당, 8B 내부 단편화

━━━ Next Fit ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  (이전 탐색 위치가 리스트 시작이라고 가정)
  처음부터 탐색 → 32B >= 24B → 32B 선택
  |
  ㄴ First Fit과 동일한 결과
  ㄴ 잔여 8B, split 불가, 32B 전체 할당

  (만약 이전 위치가 64B 이후였다면)
  → 24B >= 24B → 24B 선택 → 잔여 0B, 딱 맞는 할당

━━━ Best Fit ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  전체 탐색해서 요청(24B)에 가장 가까운 블록 선택
  |
  ㄴ 32B: 잔여 8B
  ㄴ 64B: 잔여 40B
  ㄴ 24B: 잔여 0B ← 가장 가까움
  ㄴ 128B: 잔여 104B
  |
  ㄴ 24B 블록 선택 → 잔여 0B → split 필요 없음
  ㄴ 완벽한 매칭, 단편화 없음

━━━ 결과 비교 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  정책       | 선택 블록 | 잔여   | split | 단편화
  -----------|----------|--------|-------|--------
  First Fit  | 32B      | 8B     | 불가  | 내부 8B
  Next Fit   | 32B      | 8B     | 불가  | 내부 8B
  Best Fit   | 24B      | 0B     | 없음  | 없음

  Best Fit이 이 케이스에서는 가장 효율적이다.
  하지만 항상 그런 건 아니다 — 작은 잔여 조각이 쌓이면
  그것들이 외부 단편화로 이어질 수도 있다.
```
