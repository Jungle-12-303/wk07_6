# Q1. VA와 PA의 정체 — '번역'이지 '저장'이 아니다

> CSAPP 9.1 | 1부. 주소와 공간 | 기본

## 질문

1. VA(가상주소)와 PA(물리주소)의 차이를 설명하시오.
2. "가상 메모리는 저장 공간이 아니라 주소 변환 체계다"를 근거와 함께 서술하시오.
3. [검증] 프로그램이 포인터 `0x4000`을 읽을 때, 이 값이 물리 메모리의 어떤 주소에 대응되는지 결정하는 주체와 과정을 설명하시오.

## 답변

### 최우녕

> VA(가상주소)와 PA(물리주소)의 차이를 설명하시오.

VA(Virtual Address)는 CPU가 명령어를 실행할 때 발행하는 논리 주소다.
실제로 값을 저장하는 메모리 공간이 아니라, 커널이 설정한 페이지 테이블과 연계하여
CPU가 실제 값이 저장된 물리 메모리를 가리키기 위해 사용하는 주소 체계다.

PA(Physical Address)는 DRAM 셀을 직접 지정하는 주소로, 시스템 전체에서 유일하다.

x86-64 기준 프로세스마다 독립된 48비트 가상 주소 공간(0 ~ 2⁴⁸−1)을 가지며,
프로세스 간 물리 주소 공간을 독립적으로 추상화하여 사용한다.

참고 — 주소와 저장소는 다르다:
주소 1개는 메모리의 **1바이트 위치**를 가리키는 번호표다(바이트 주소 지정).
주소 자체를 저장하려면 64비트 시스템에서 **8바이트**가 필요하다.
그래서 C에서 `sizeof(int *)` = 8이다. 포인터 변수에 주소(번호)를 담는 데 8바이트를 쓰는 것이다.
x86-64는 레지스터가 64비트(8바이트)이지만, 실제 주소로 의미 있는 건 하위 48비트뿐이고
상위 16비트는 부호 확장으로 채운다(canonical address).

> "가상 메모리는 저장 공간이 아니라 주소 변환 체계다"를 근거와 함께 서술하시오.

데이터를 실제로 담는 저장 공간은 DRAM과 디스크(스왑)뿐이다.
가상 메모리는 그 자체에 바이트를 담는 셀이 없으며,
추상화된 논리 주소를 물리 주소에 연결하기 위한 식별자 역할만 수행한다.

가상 주소에 접근하면 커널이 관리하는 페이지 테이블을 통해 실제 물리 주소를 알아낼 수 있고,
CPU는 알아낸 물리 주소에 접근해 값을 읽거나 쓴다.
즉 가상 메모리는 "VA → PA 변환 규칙"이지, 데이터를 저장하는 공간이 아니다.

> [검증] 프로그램이 포인터 `0x4000`을 읽을 때, 이 값이 물리 메모리의 어떤 주소에 대응되는지 결정하는 주체와 과정을 설명하시오.

```text
전제: x86-64, 4KB 페이지(오프셋 12비트), 4단계 페이지 테이블

━━━ 1. VA 분해 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  VA = 0x0000_0000_0000_4000

  [    VPN = 0x4     |   VPO = 0x000    ]
    어떤 페이지?        페이지 안 어디?

━━━ 2. 주소 변환 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  CPU: mov 명령 → VA 0x4000 → MMU 전달
  |
  MMU: TLB에서 VPN=4 검색
  |
  ㄴ HIT  → PPN 즉시 획득 → [4. PA 조합]으로
  ㄴ MISS → 페이지 테이블 워크

━━━ 3. 페이지 테이블 워크 ━━━━━━━━━━━━━━━━━━━━━━━

  CR3 (PML4 베이스 주소)
  |
  ㄴ 1단계  PML4   [47:39] = 0  → PDPT
     ㄴ 2단계  PDPT   [38:30] = 0  → PD
        ㄴ 3단계  PD     [29:21] = 0  → PT
           ㄴ 4단계  PT     [20:12] = 4  → PTE 획득

  ※ 0x4000은 작은 주소 → 상위 3단계 인덱스 모두 0
    실제 분기는 PT 4번 엔트리뿐

  PTE 확인
  |
  ㄴ Valid=1 → PPN 획득 (예: 0xA3), TLB 캐싱 → [4. PA 조합]으로
  ㄴ Valid=0 → Page Fault
     |
     커널 폴트 핸들러
     ㄴ DRAM 프레임 할당 (없으면 victim evict)
     ㄴ 디스크에서 페이지 로드
     ㄴ PTE 갱신 (PPN 기록 + Valid=1)
     ㄴ TLB 캐싱
     ㄴ 같은 mov 명령 재실행 → 이번엔 Valid=1

━━━ 4. PA 조합 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  PA = (PPN << 12) | VPO
     = (0xA3 << 12) | 0x000
     = 0xA3000

  [    PPN = 0xA3    | PPO = VPO = 0x000 ]
    물리 페이지 번호     오프셋 (그대로)

━━━ 5. 데이터 접근 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  PA 0xA3000
  ㄴ L1 HIT → 반환 (~4 사이클)
  ㄴ L2 HIT → 반환 (~10 사이클)
  ㄴ L3 HIT → 반환 (~40 사이클)
  ㄴ MISS   → DRAM 접근 (~200 사이클)

결정 주체: HW = MMU (TLB + 페이지 워커)
          SW = 커널 (폴트 핸들러)
```
