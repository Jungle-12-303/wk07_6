# Q10. VPN + Offset 분해 — 비트 연산으로 이해하기

> CSAPP 9.6 | 4부. 주소 변환 | 검증

## 질문

1. VA를 VPN과 VPO로 분해하는 공식을 비트 연산으로 표현하시오.
2. VPO = PPO인 이유를 설명하시오.
3. [검증] 14-bit VA, 12-bit PA, 페이지 크기 64B일 때 VA `0x03D4`의 VPN, VPO, 그리고 TLB 인덱스 구조를 구하시오.

## 답변

### 최우녕

> VA를 VPN과 VPO로 분해하는 공식을 비트 연산으로 표현하시오.

페이지 크기가 2^p 바이트이면 오프셋(VPO)은 하위 p비트, 페이지 번호(VPN)는 나머지 상위 비트다.

```c
VPO = VA & ((1 << p) - 1)   // 하위 p비트 마스킹
VPN = VA >> p                // 상위 비트만 남김
```

`&`는 비트 AND 연산이다. 둘 다 1인 비트만 살린다.
`(1 << p) - 1`은 하위 p비트만 1인 마스크를 만든다.
예를 들어 p=12이면 `(1 << 12) - 1 = 4095 = 0xFFF = 0b 1111 1111 1111`이다.
VA와 AND하면 하위 12비트(오프셋)만 꺼내지고 나머지는 0이 된다.

`>>`는 오른쪽 시프트. `VA >> 12`는 하위 12비트를 버리고
나머지를 오른쪽으로 밀어서 VPN만 남긴다.

예: 4KB 페이지 (p = 12)
```
VPO = VA & 0xFFF
VPN = VA >> 12
```

실제 계산:
```
VA     = 0x4000 = 0100 0000 0000 0000
0xFFF  =          0000 1111 1111 1111  (하위 12비트만 1)
AND    =          0000 0000 0000 0000  = 0x000 (VPO)

VA >> 12 = 0100 = 0x4 (VPN)
```

> VPO = PPO인 이유를 설명하시오.

페이지는 가상 주소와 물리 주소를 "페이지 단위"로 매핑한다.
변환하는 건 "몇 번째 페이지인가"(VPN → PPN)뿐이고,
"페이지 안에서 몇 번째 바이트인가"(오프셋)는 바뀔 이유가 없다.

가상 페이지의 0번째 바이트는 물리 페이지의 0번째 바이트에 대응하고,
100번째 바이트는 100번째에 대응한다. 페이지 내부 구조가 그대로 유지되므로
VPO = PPO다. 오프셋은 변환 없이 PA에 그대로 붙는다.

> [검증] 14-bit VA, 12-bit PA, 페이지 크기 64B일 때 VA `0x03D4`의 VPN, VPO, 그리고 TLB 인덱스 구조를 구하시오.

```text
전제: 14-bit VA, 12-bit PA, 페이지 크기 64B = 2^6

━━━ 1. 비트 구조 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  오프셋 비트 수 = log2(64) = 6비트
  VPN 비트 수 = 14 - 6 = 8비트
  PPN 비트 수 = 12 - 6 = 6비트

  VA 14비트:
  [  VPN (8bit)  |  VPO (6bit)  ]

  PA 12비트:
  [  PPN (6bit)  |  PPO (6bit)  ]

━━━ 2. VA 0x03D4 분해 ━━━━━━━━━━━━━━━━━━━━━━━━━━

  0x03D4 = 0b 00 0011 1101 0100
                ────────────── ──────────
                VPN (8bit)     VPO (6bit)

  VPN = 0x03D4 >> 6
      오른쪽으로 6비트 밀기 = 하위 6비트 버림
      = 0b 00001111 = 0x0F = 15

  VPO = 0x03D4 & 0x3F
      0x3F = 0b 111111 (하위 6비트만 1인 마스크)
      AND하면 하위 6비트만 남음
      = 0b 010100 = 0x14 = 20

  ㄴ 15번 페이지의 20번째 바이트

━━━ 3. TLB 인덱스 구조 ━━━━━━━━━━━━━━━━━━━━━━━━━

  TLB가 4-way set associative, 16 set이라 가정하면:

  VPN 8비트를 나눔:
  [  TLBT (4bit)  |  TLBI (4bit)  ]
     태그              셋 인덱스

  VPN = 0x0F = 0b 00001111
  ㄴ TLBT = 0b 0000 = 0x0
  ㄴ TLBI = 0b 1111 = 0xF = 15번 셋

  TLB 조회: 15번 셋의 4개 way에서 태그 0x0 매칭 검색
```
