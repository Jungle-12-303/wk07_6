# Q19. 내부 단편화 vs 외부 단편화 — 숫자로 체감하기

> CSAPP 9.9 | 8부. malloc | 기본

## 질문

1. 내부 단편화와 외부 단편화의 차이를 정의하시오.
2. 각각을 줄이기 위한 전략을 서술하시오.
3. [검증] 힙에 [16B할당][48B가용][16B할당][32B가용] 상태에서 malloc(60)이 실패하는 이유를 내부/외부 단편화로 설명하시오.

## 답변

### 최우녕

> 내부 단편화와 외부 단편화의 차이를 정의하시오.

내부 단편화(Internal Fragmentation):
할당된 블록 안에서 실제로 쓰지 않는 부분이 낭비되는 것이다.
예를 들어 13바이트를 요청했는데 정렬 때문에 16바이트 블록을 할당하면
3바이트가 패딩으로 낭비된다. 블록 "안쪽"의 낭비라서 내부 단편화다.

외부 단편화(External Fragmentation):
빈 공간의 총합은 충분하지만 연속되지 않아서 큰 블록을 할당할 수 없는 상태다.
예를 들어 48B짜리 빈 공간과 32B짜리 빈 공간이 따로 있으면
합치면 80B인데 60B짜리 연속 블록은 만들 수 없다.
블록 "바깥"에서 조각난 빈 공간이 문제라서 외부 단편화다.

> 각각을 줄이기 위한 전략을 서술하시오.

내부 단편화 줄이기:
ㄴ 정렬 단위를 줄이면 패딩이 줄어든다 (하지만 성능 트레이드오프)
ㄴ 요청 크기에 딱 맞는 size class를 쓴다 (Segregated List)
ㄴ 작은 블록 전용 allocator 사용 (slab allocator 등)

외부 단편화 줄이기:
ㄴ Coalescing: 인접한 빈 블록을 합쳐서 큰 블록으로 만든다
ㄴ Best Fit: 요청 크기에 가장 가까운 빈 블록을 선택해서 잔여 조각을 최소화
ㄴ Compaction: 사용 중인 블록을 한쪽으로 모아서 빈 공간을 연속으로 만든다
   (GC가 있는 언어에서 가능, C의 malloc에서는 포인터가 바뀌므로 불가능)

> [검증] 힙에 [16B할당][48B가용][16B할당][32B가용] 상태에서 malloc(60)이 실패하는 이유를 내부/외부 단편화로 설명하시오.

```text
━━━ 현재 힙 상태 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [16B 할당][48B 가용][16B 할당][32B 가용]

  총 가용 공간 = 48 + 32 = 80B
  요청 크기 = 60B

━━━ 왜 실패하는가 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  80B가 비어있지만 연속된 60B 블록이 없다.
  |
  ㄴ 48B 블록 → 60B 부족 (12B 모자람)
  ㄴ 32B 블록 → 60B 부족 (28B 모자람)
  ㄴ 두 블록 사이에 16B 할당 블록이 끼어있어서 합칠 수 없음

  이것이 외부 단편화다.
  빈 공간의 총합은 충분하지만 조각나 있어서 할당 불가능.

━━━ 만약 내부 단편화도 있다면 ━━━━━━━━━━━━━━━━━

  실제로는 헤더(8B) + 정렬 패딩까지 포함하면
  malloc(60) → 필요한 실제 블록 크기 = 60 + 8(헤더) = 68B
  정렬로 올림하면 72B 또는 80B가 될 수 있다.

  48B 가용 블록은 72B 요청에 더더욱 부족하다.

━━━ 해결하려면 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ㄴ 16B 할당 블록이 free되면 → coalesce로 48+16+32 = 96B 연속 확보
  ㄴ 애초에 배치 정책을 잘 했다면 조각이 덜 났을 수 있음
```
